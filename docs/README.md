## 📝 文件命名规范

所有文档文件遵循统一的命名规范：

### 格式：`{序号}-{功能模块}-{具体内容}.md`

- **序号**：两位数字（01-99），用于排序
- **功能模块**：简洁的英文描述，使用连字符分隔
- **具体内容**：详细描述文档内容，使用连字符分隔
- **扩展名**：使用 `.md` 表示Markdown文档

### 示例：
- `01-project-status.md` - 项目状态文档
- `02-dual-mode-architecture.md` - 双模式架构文档
- `03-human-in-loop-implementation-plan.md` - 人机交互实现计划
